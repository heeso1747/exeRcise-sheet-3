    • Strengths:
      • Very hard to lose files with git, as it is designed to track changes to files and directories. This makes it very easy to revert to a previous version of a file or directory if something goes wrong.
      • Git is very fast and efficient. It is designed to handle large projects and is very good at managing changes to the codebase. This makes it a great tool for teams working on large projects.
    • Weaknesses:
      • Git can be complex to use. It has a lot of features and options that can be overwhelming for new users. This can make it hard to use git effectively and can lead to mistakes if you are not careful.
      • Struggles with binary (i.e.¬†non-text) files and large files. Git is designed to track changes to text files, so it can struggle with binary files like images, videos, and executables. This can make it hard to track changes to these files and can lead to problems when working with them in git.

# Calculate initial and final life expectancy for each country
life_expectancy_summary <- life_expectancy %>%
  group_by(Entity) %>%
  summarise(Initial_Life_Expectancy = first(`Life Expectancy (OECD)`),
            Final_Life_Expectancy = last(`Life Expectancy (OECD)`),
            Growth_Rate = (Final_Life_Expectancy - Initial_Life_Expectancy) / Initial_Life_Expectancy * 100) %>%
  arrange(desc(Growth_Rate))
